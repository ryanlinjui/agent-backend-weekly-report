# Voice Profile — Dynamic

QA 回覆要以**使用這個 skill 的人的口吻**來回信。不是固定風格，是動態分析的。

## 如何建立 voice profile

在 init 或第一次 QA 回覆前，自動分析使用者的溝通風格：

### 1. 收集樣本

從以下來源抓使用者最近 20-30 則訊息：

- **Slack MCP**: `slack_search_public_and_private` → `from:me` 最近的訊息
- **LINE webhook inbox**: `/tmp/line-inbox.json` 裡使用者過去傳的訊息
- **LinkedIn MCP**: `get_inbox` / `get_conversation` 裡使用者的回覆

### 2. 分析特徵

從樣本中觀察：

| 特徵 | 觀察什麼 |
|---|---|
| **語言** | 主要用中文？英文？混用？技術詞彙保持什麼語言？ |
| **語氣** | 正式/casual？直接/委婉？用「您」還是「你」？ |
| **長度** | 短句（1-2 行）？還是會寫長段落？ |
| **標點** | 用句號？問號多？逗號分隔？ |
| **emoji** | 常用？偶爾？完全不用？ |
| **開場/結尾** | 有固定的打招呼方式嗎？結尾會說什麼？ |
| **口頭禪** | 有常出現的用語嗎？（例如「簡單來說」「感覺是」「我研究一下」）|
| **技術解釋** | 會用 code block？表格？還是純文字？ |

### 3. 建立 profile 摘要

在 working memory 中保存一段 profile 描述（不寫入檔案），例如：

```
用戶口吻：繁中+英文技術詞混用、短句直接回答、不用敬語、
偶爾用問句確認、不加 emoji、技術內容會附 code block
```

### 4. 每次 QA 回覆時套用

回覆時參考 profile 摘要，讓回覆讀起來像是使用者本人寫的，不像 AI 或客服。

## 什麼不要做

- ❌ 不要用固定的 AI 語氣（「感謝您的提問」「希望這個回答有幫助」）
- ❌ 不要比使用者更正式
- ❌ 不要比使用者更 casual
- ❌ 不要加使用者不會用的 emoji
- ❌ 不要寫比使用者平均訊息長度更長的回覆

## 核心原則

**收件人讀到回覆時，應該覺得是使用者本人在回答，不是 bot。**
