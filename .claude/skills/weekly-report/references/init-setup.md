Auto-check all services and install missing MCP servers on first run.

## Required config (.env)

| Variable | Purpose | How to verify |
|---|---|---|
| `GITHUB_USERNAME` | GitHub user to query | `gh auth status` returns OK |
| `REPORT_RECIPIENTS` | Email addresses | Non-empty |
| `REPORT_WINDOW_DAYS` | Days to look back (default: 7) | Non-empty |
| `EMAIL_USER` | Email account (any provider) | Non-empty |
| `EMAIL_PASSWORD` | App Password for IMAP/SMTP | SMTP test succeeds |
| `SLACK_USER_TOKEN` | Slack User Token for search | `curl` auth.test returns OK |
| `LINE_CHANNEL_ACCESS_TOKEN` | LINE Messaging API | `curl` bot/info returns OK |

## 0b: Auto-install missing MCP servers

### Detect environment

- Claude Code: `claude --version` succeeds Ôćĺ use `claude mcp add-json`
- Claude Desktop: `~/Library/Application Support/Claude/claude_desktop_config.json` exists Ôćĺ edit with Write tool

## 0c: Email setup (if EMAIL_PASSWORD missing or auth fails)

Most email providers (Gmail, Outlook, Yahoo, iCloud) require an App Password for IMAP/SMTP.

**We never store the user's main password.** Only the App Password (a separate, revocable token generated by the provider).

### Helper: open URL in user's default browser

```bash
open_url() {
  case "$(uname -s)" in
    Darwin) open "$1" ;;
    Linux)  xdg-open "$1" 2>/dev/null || echo "Please open: $1" ;;
    MINGW*|MSYS*) start "$1" 2>/dev/null || echo "Please open: $1" ;;
    *)      echo "Please open: $1" ;;
  esac
}
```

### Step 1: Open login page via Chrome DevTools MCP Ôćĺ user logs in

Use Chrome DevTools MCP to open the provider's login page. The user logs in and does 2FA in that browser window.

Detect provider from `EMAIL_USER` domain:

| Domain | Login URL |
|---|---|
| gmail.com / Google Workspace | `https://accounts.google.com/signin` |
| outlook.com / hotmail.com | `https://login.live.com` |
| yahoo.com | `https://login.yahoo.com` |
| icloud.com / me.com | `https://appleid.apple.com` |

```

```

Print:
```

   1. Log in to {EMAIL_USER}
   2. Complete phone/2FA verification if prompted
   Then say "ok".
```

**Wait for "ok".** User only does login + phone verification. Nothing else.

### Step 2: Skill auto-navigates to 2FA settings

After user confirms login, skill uses Chrome DevTools MCP to check if 2FA is enabled:

```
Chrome DevTools MCP: navigate_page Ôćĺ https://myaccount.google.com/signinoptions/two-step-verification
Chrome DevTools MCP: take_snapshot Ôćĺ check if 2FA is "On" or "Off"
```

- If 2FA is already ON Ôćĺ skip to Step 3
- If 2FA is OFF Ôćĺ print: `­čîÉ Please enable 2FA on this page (phone verification), then say "ok".` Ôćĺ wait Ôćĺ re-check

### Step 3: Skill auto-creates App Password

After 2FA is confirmed ON, skill navigates to App Passwords and creates one automatically:

```
Chrome DevTools MCP: navigate_page Ôćĺ https://myaccount.google.com/apppasswords
Chrome DevTools MCP: take_snapshot Ôćĺ find the "App name" input field
Chrome DevTools MCP: fill Ôćĺ type "weekly-report"
Chrome DevTools MCP: click Ôćĺ "Create" button
Chrome DevTools MCP: take_snapshot Ôćĺ read the generated 16-character App Password from the page
```

Extract the App Password from the page content (it's displayed once after creation).

**No user interaction needed for this step.** Skill does it all.

For Outlook/Yahoo/iCloud: adapt the navigation steps to each provider's App Password UI.

## User interaction principle

User only interacts for **three things** during the entire `/weekly-report` flow:
1. **First-run setup** ÔÇö connect MCPs + paste App Password (once only)
2. **Logging in** ÔÇö if email/MCP sessions expire (rare)
3. **Approving the report** ÔÇö review draft Ôćĺ send

Everything else is automated.
